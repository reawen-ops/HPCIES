# 节点数获取流程图

## 简化流程

```
CSV 文件 → 解析数据 → 找最大负载 → 计算节点数 → 存储 → 显示
```

## 详细流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     1. 用户上传 CSV 文件                          │
│                                                                 │
│  前端: Welcome.tsx                                              │
│  ├─ 用户选择文件                                                │
│  ├─ 点击"提交配置并开始分析"                                      │
│  └─ 调用 uploadHistory(file)                                   │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│                     2. 后端解析 CSV                              │
│                                                                 │
│  后端: others.py → upload_history()                             │
│  ├─ 读取文件内容                                                │
│  ├─ 使用 pandas 解析 CSV                                        │
│  └─ 提取所有数据行                                              │
│                                                                 │
│  示例数据:                                                      │
│  ┌──────────┬──────┬────────────────┐                          │
│  │ 日期     │ 小时 │ CPU核时使用量   │                          │

└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│                  3. 自动推断节点数                               │
│                                                                 │
│  推断算法:                                                      │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ Step 1: 找出最大负载                                      │  │

│                                                                 │
│  代码位置: backend/src/app/api/routes/others.py (第 195-205 行) │
```

## 关键点总结

### 推断公式
```
总节点数 = ⌊(最大负载 × 1.2) / 32⌋
```

### 数据存储
```
user_profile 表
├─ user_id: 用户ID
├─ node_count: 总节点数 ← 这里存储
├─ core_per_node: 每节点核心数
└─ has_history: 是否有历史数据
```

### API 端点
```
POST /api/upload-history  → 上传CSV，推断并存储节点数
GET  /api/stats           → 获取节点数并显示
```

### 代码位置
```
推断逻辑: backend/src/app/api/routes/others.py (第 195-205 行)
存储逻辑: backend/src/app/api/routes/others.py (第 207-211 行)
读取逻辑: backend/src/app/api/routes/others.py (第 40-48 行)
显示组件: frontend/src/components/common/StatisticsInfo.tsx
```

## 示例计算

### 示例 1: 小型集群
```
CSV 数据最大负载: 500
↓
估算总核心数: 500 × 1.2 = 600
↓
节点数: 600 ÷ 32 = 18.75 → 18 个节点
```

### 示例 2: 中型集群
```
CSV 数据最大负载: 1600
↓
估算总核心数: 1600 × 1.2 = 1920
↓
节点数: 1920 ÷ 32 = 60 个节点
```

### 示例 3: 大型集群
```
CSV 数据最大负载: 3200
↓
估算总核心数: 3200 × 1.2 = 3840
↓
节点数: 3840 ÷ 32 = 120 个节点
```

## 调整参数

如果推断不准确，可以修改：

### 1. 每节点核心数
```python
# backend/src/app/api/routes/others.py (第 203 行)
core_per_node = 32  # 改为 64, 128 等
```

### 2. 余量系数
```python
# backend/src/app/api/routes/others.py (第 201 行)
estimated_total_cores = int(max_load * 1.2)  # 改为 1.5, 2.0 等
```

### 3. 直接修改数据库
```sql
UPDATE user_profile 
SET node_count = 50 
WHERE user_id = 1;
```
