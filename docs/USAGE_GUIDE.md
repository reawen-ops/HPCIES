# HPCIES 使用指南

## 快速开始

### 1. 启动后端服务

```bash
cd backend/src
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 启动前端服务

```bash
cd frontend
npm install
npm run dev
```

前端将在 `http://localhost:5173` 启动

### 3. 访问系统

在浏览器中打开 `http://localhost:5173`

## 完整使用流程

### 步骤 1: 注册账号

1. 访问系统首页，自动跳转到注册页面
2. 输入用户名（2-64 字符）
3. 输入密码（6-128 字符）
4. 点击"注册"按钮

### 步骤 2: 登录系统

1. 注册成功后跳转到登录页面
2. 输入用户名和密码
3. 点击"登录"按钮
4. 登录成功后进入主页面

### 步骤 3: 上传历史数据

首次登录会弹出欢迎配置页面：

1. 点击上传区域或拖拽 CSV 文件
2. 确认文件已选择
3. 点击"提交配置并开始分析"

**CSV 文件格式要求：**

格式 1（推荐）：
```csv
日期,小时,CPU核时使用量
2025-12-01,0,150.5
2025-12-01,1,145.2
2025-12-01,2,140.8
...
```

格式 2：
```csv
2025-12-01 00:00:00,150.5
2025-12-01 01:00:00,145.2
2025-12-01 02:00:00,140.8
...
```

**注意事项：**
- 文件编码必须是 UTF-8
- 至少需要 24 小时的连续数据
- 数据越多，预测越准确（建议至少 1 个月）
- 数据应该是每小时一条记录

### 步骤 4: 查看预测结果

上传成功后，系统会自动：
1. 推断集群配置（节点数、每节点核心数）
2. 生成节点状态矩阵
3. 显示主界面

主界面包含：

#### 4.1 统计信息卡片
- 总节点数
- 运行节点数
- 今日预计节能百分比
- 今日预计任务数

#### 4.2 预测曲线图
- 选择日期查看该日的 24 小时预测
- 对比全开模式和节能模式的利用率
- 查看节能策略建议
- 查看节能效果评估
- 查看任务影响分析

**预测曲线功能：**
- 日期选择：选择要查看的日期
- 预测范围：今日/未来3天/未来7天
- 显示模式：对比模式/仅全开/仅节能
- 更新频率：实时/每小时/每6小时/每日
- 重新计算：手动触发预测更新

#### 4.3 节点矩阵
- 可视化展示所有节点状态
- 绿色：运行中
- 黄色：待休眠
- 灰色：休眠中

#### 4.4 AI 对话助手
- 输入问题获取节能建议
- 查看历史对话记录
- 清空对话历史

### 步骤 5: 分析预测结果

#### 节能策略
- **建议休眠时段**：系统识别的低负载时段
- **节点状态建议**：
  - 必须运行：50% 节点保持运行
  - 待休眠：16% 节点准备休眠
  - 休眠：34% 节点进入休眠
- **提前唤醒时间**：提前 15 分钟唤醒节点

#### 节能效果
- **预计节能**：相比全开模式的节能百分比
- **节省核时**：每天节省的核心小时数
- **节省电力**：每天节省的电力（kWh）

#### 任务影响
- **预计任务延迟**：任务可能的延迟时间
- **队列积压风险**：任务队列积压的风险等级
- **紧急响应能力**：应对突发任务的能力评级

## 常见问题

### Q1: 上传 CSV 失败怎么办？

**可能原因：**
1. 文件格式不正确
2. 文件编码不是 UTF-8
3. 数据列缺失或列名错误
4. 数据中包含非法字符

**解决方法：**
1. 检查 CSV 文件格式是否符合要求
2. 使用文本编辑器另存为 UTF-8 编码
3. 确保包含必需的列
4. 清理数据中的特殊字符

### Q2: 预测数据显示"历史数据不足"？

**原因：**
系统需要目标日期前至少 24 小时的历史数据才能进行预测。

**解决方法：**
1. 上传更多历史数据
2. 选择有足够历史数据的日期进行预测
3. 确保数据是连续的，没有大的时间间隔

### Q3: 预测服务暂时不可用？

**原因：**
LSTM API 服务可能暂时无法访问。

**解决方法：**
1. 检查网络连接
2. 稍后重试
3. 联系管理员检查 API 服务状态

### Q4: 登录过期怎么办？

**原因：**
Token 有效期为 24 小时，超时后需要重新登录。

**解决方法：**
1. 点击退出登录
2. 重新输入用户名和密码登录
3. 系统会自动保存新的 Token

### Q5: 预测结果不准确？

**可能原因：**
1. 历史数据量不足
2. 历史数据质量差（缺失值、异常值）
3. 负载模式发生变化

**改进方法：**
1. 上传更多历史数据（建议至少 3 个月）
2. 清理异常数据
3. 定期更新历史数据
4. 考虑重新训练 LSTM 模型

### Q6: 如何理解节点状态？

- **运行中（绿色）**：节点正在运行，处理任务
- **待休眠（黄色）**：节点准备进入休眠，完成当前任务后休眠
- **休眠中（灰色）**：节点已休眠，节省能源

### Q7: 节能策略如何应用到实际集群？

系统提供的是建议策略，实际应用需要：
1. 导出策略配置
2. 根据集群管理系统调整
3. 设置自动化脚本
4. 监控实际效果
5. 根据反馈调整参数

## 最佳实践

### 数据准备
1. 收集至少 3 个月的历史数据
2. 确保数据完整性和准确性
3. 定期更新历史数据
4. 记录特殊事件（如系统维护、大规模任务）

1. 每天查看次日预测
2. 对比预测和实际使用情况

3. 监控任务延迟和队列情况

3. 监控 API 调用频率
